Livewire Installations and AdminLTE Mastering

Last updated: August 9th, 2023

Installations

Step One - Livewire Install

For Install Livewire enter this command

composer require livewire/livewire

php artisan livewire:publish --config
php artisan livewire:publish --assets

After publishing config and asset if you do not configure the path in config/livewire.php -
'asset_url' => null to 'asset_url' => env('APP_URL') and 'app_url' => null to 'app_url' => env('APP_URL') by default it will not find the livewire path by entering (http://localhost/livewire_admin/public/dashboard) this link .Its only work in server section php artisan server - (http://127.0.0.1:8000/dashboard)

works fine in - http://127.0.0.1:8000/dashboard

Not works in - http://localhost/livewire_admin/public/dashboard

And If you change the asset path in config/livewire.php -
'asset_url' => null to 'asset_url' => env('APP_URL') and 'app_url' => null to 'app_url' => env('APP_URL') then it will not find the livewire path by entering (http://127.0.0.1:8000/dashboard) this link .Its only work in public section (http://localhost/livewire_admin/public/dashboard)

works fine in - http://localhost/livewire_admin/public/dashboard

Not works in - http://127.0.0.1:8000/dashboard

Step Three -Make Livewire Components and Load the View into "backend/DashboardController.php"
For make Component enter this command

php artisan make:livewire DashboardComponent
this will create two files - one is livewire/DashboardComponent.php and another is view livewire/dashboard-component.blade.php

For make Controller enter this command

php artisan make:controller DashboardController -r

Also Create a view called backend/dashboard.blade.php

Step Four: Disabling Browser Reload with Turbolinks

For Install Turbolinks enter this command

npm install --save turbolinks
npm run dev

Add the code into resources/js/app.js

import Turbolinks from "turbolinks"; Turbolinks.start()

php artisan optimize:clear

or

const Turbolinks = require("turbolinks") Turbolinks.start()

php artisan optimize:clear
